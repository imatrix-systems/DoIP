# DoIP Server Build Guide

Developer reference for building the DoIP server on Ubuntu and cross-compiling for the Torizon embedded target.

## Platform Overview

The DoIP server supports two build platforms:

| Platform | Target | Compiler | Optimization | libc | Define |
|----------|--------|----------|--------------|------|--------|
| `ubuntu` (default) | x86-64 Linux PC | Native `gcc` | `-O2` | glibc | `PLATFORM_UBUNTU` |
| `torizon` | Toradex Verdin iMX8MPlus | QConnect SDK `arm-linux-gcc` | `-Os` | musl | `PLATFORM_TORIZON` |

The Torizon target runs Torizon Linux (not containerized) on the Toradex Verdin iMX8MPlus SoM. The QConnect SDK provides a Buildroot-based toolchain with GCC 6.4.0 targeting ARM Cortex-A7 hard-float with musl libc — the same toolchain used by Fleet-Connect-1.

## Prerequisites

### Ubuntu Build (development, testing, CI)

```bash
sudo apt install build-essential
gcc --version    # GCC 7+ for C11 support
make --version   # GNU Make
```

### Torizon Cross-Build

The QConnect SDK must be installed at `/opt/qconnect_sdk_musl/`:

```
/opt/qconnect_sdk_musl/
├── bin/
│   ├── arm-linux-gcc          # C compiler (-> toolchain-wrapper)
│   ├── arm-linux-g++          # C++ compiler
│   ├── arm-linux-ar           # Archiver
│   ├── arm-linux-ld           # Linker
│   └── arm-linux-ranlib       # Ranlib
└── arm-buildroot-linux-musleabihf/
    └── sysroot/               # Target root filesystem (headers + libs)
```

Verify the toolchain:

```bash
/opt/qconnect_sdk_musl/bin/arm-linux-gcc --version
# arm-linux-gcc.br_real (Buildroot 2018.02.6-g926ee31) 6.4.0

file /opt/qconnect_sdk_musl/bin/arm-linux-gcc
# ... symbolic link to toolchain-wrapper
```

## Building

### Ubuntu (Native)

```bash
cd DoIP_Server

# Build everything (server + 3 test binaries)
make

# Build server only
make doip-server

# Verbose build (see commands)
make V=1
```

### Torizon (Cross-Compile)

```bash
# Default: uses QConnect SDK at /opt/qconnect_sdk_musl/
make PLATFORM=torizon

# Explicit toolchain path (if SDK is elsewhere)
make PLATFORM=torizon CROSS_COMPILE=/path/to/arm-linux-

# Verify the binary is ARM
file doip-server
# doip-server: ELF 32-bit LSB executable, ARM, EABI5, version 1 (SYSV), dynamically linked, ...
```

### Build Verification

```bash
# Invalid platform produces clear error
make PLATFORM=invalid
# *** PLATFORM must be 'ubuntu' or 'torizon' (got 'invalid').  Stop.

# Clean build
make clean && make PLATFORM=ubuntu
# Zero warnings expected

make clean && make PLATFORM=torizon
# Zero warnings expected
```

## What Changes Per Platform

### Compile-Time Differences

| Feature | Ubuntu (`PLATFORM_UBUNTU`) | Torizon (`PLATFORM_TORIZON`) |
|---------|---------------------------|------------------------------|
| **Optimization** | `-O2` (performance) | `-Os` (size) |
| **`explicit_bzero`** | System-provided (glibc) | Volatile byte-loop fallback (musl may lack it) |
| **SSH keygen script** | `ssh-keygen` (OpenSSH) | `dropbearkey` (Dropbear) |
| **HTTPS registration script** | `curl -X POST` | `wget --post-data` |
| **Secret shredding** | `shred -u` (GNU coreutils) | `dd if=/dev/urandom` (BusyBox) |
| **CLI platform display** | "Ubuntu" | "Torizon" |

### Runtime Behavior

The server binary behaves identically on both platforms. The only runtime difference is which embedded scripts are generated by the `generate-scripts` command — they contain platform-appropriate tool invocations.

## Compiler Flags

Both platforms use:

```
-Wall -Wextra -Wpedantic -g -std=c11 -DPLATFORM_<name>
```

Plus:
- `-O2` for Ubuntu, `-Os` for Torizon
- `-Iinclude` for headers
- `-lpthread` for POSIX threads

The C11 standard is used with `_POSIX_C_SOURCE 200809L` for `strtok_r` and POSIX file operations.

## Installation

### Development Install

```bash
# Install to /usr/sbin/ (default PREFIX=/usr)
sudo make install

# Install to custom prefix
make install PREFIX=/usr/local DESTDIR=/tmp/staging
```

### Production Install (Torizon)

```bash
# Cross-compile
make clean && make PLATFORM=torizon

# Stage for deployment
make install PLATFORM=torizon DESTDIR=/tmp/doip-staging PREFIX=/usr

# Copy to device (example using FC-1 network)
scp -P 22222 /tmp/doip-staging/usr/sbin/doip-server root@192.168.7.1:/usr/sbin/
scp -P 22222 doip-server.conf root@192.168.7.1:/etc/doip/
```

### Installed Files

```
$(DESTDIR)$(PREFIX)/sbin/doip-server              # Server binary
$(DESTDIR)$(PREFIX)/sbin/phonehome-keygen.sh       # Key generation script
$(DESTDIR)$(PREFIX)/sbin/phonehome-register.sh     # Registration script
$(DESTDIR)$(PREFIX)/sbin/phonehome-connect.sh      # Tunnel script
$(DESTDIR)/etc/phonehome/phonehome.conf            # Example phone-home config
```

### Cross-Compiled Binary Testing

Cross-compiled Torizon binaries cannot run tests on the build host. To verify:

```bash
# 1. Check binary format
file doip-server
# ELF 32-bit LSB executable, ARM, EABI5 ...

# 2. Check for unresolved symbols (requires target sysroot)
/opt/qconnect_sdk_musl/bin/arm-linux-readelf -d doip-server | grep NEEDED
# NEEDED: libpthread.so.0, libc.so

# 3. Deploy to device and test
scp -P 22222 doip-server root@192.168.7.1:/tmp/
ssh -p 22222 root@192.168.7.1 "/tmp/doip-server -c /etc/doip/doip-server.conf -v"
```

## Source Architecture

```
src/main.c              # Entry point, arg parsing, daemon, CLI loop, UDS dispatch
src/config.c            # Config file parser (key=value)
src/doip.c              # DoIP protocol encoding/decoding
src/doip_server.c       # TCP/UDP server, client threading, connection management
src/doip_log.c          # Structured logging with file rotation
src/cli.c               # Interactive CLI (status, config, transfer, generate-scripts)
src/script_gen.c        # Embedded scripts as C strings, file writer
src/phonehome_handler.c # UDS RoutineControl 0xF0A0 handler
src/hmac_sha256.c       # Standalone SHA-256 + HMAC-SHA256 (no OpenSSL)
```

### Threading Model

```
main thread        -> CLI loop (select on stdin, 1s timeout) or sleep loop
  |
  +-- TCP accept thread (doip_server_start)
  |     +-- per-client handler threads (up to max_tcp_connections)
  |
  +-- UDP handler thread (doip_server_start)
```

Shared state is protected by:
- `g_transfer_mutex` — blob transfer state (SID 0x34/0x36/0x37)
- `server.clients_mutex` — client count and connection slots
- `g_running` (`volatile sig_atomic_t`) — shutdown signal from Ctrl+C or CLI `quit`

The main loop snapshots both `g_transfer_mutex`-protected fields and `clients_mutex`-protected `num_clients` under their respective locks before passing them to CLI display functions. CLI commands never acquire mutexes directly.

### Platform-Conditional Code

All platform differences are gated by `#ifdef PLATFORM_TORIZON`:

| File | Conditional | Purpose |
|------|-------------|---------|
| `hmac_sha256.c` | `#ifdef PLATFORM_TORIZON` | `explicit_bzero` fallback for musl |
| `phonehome_handler.c` | `#ifdef PLATFORM_TORIZON` | `explicit_bzero` fallback for musl |
| `script_gen.c` | `#ifdef PLATFORM_TORIZON` | Platform-specific embedded scripts |
| `cli.c` | `#ifdef PLATFORM_TORIZON` | Platform name in status display |

## Troubleshooting Build Issues

**QConnect SDK not found:**
```
make: /opt/qconnect_sdk_musl/bin/arm-linux-gcc: No such file or directory
```
Install the QConnect SDK or specify the toolchain path:
```bash
make PLATFORM=torizon CROSS_COMPILE=/your/path/arm-linux-
```

**C11 features not supported:**
```
error: 'for' loop initial declarations are only allowed in C99 or C11 mode
```
Your GCC is too old. The server requires GCC 5+ for C11 support.

**Missing pthread:**
```
undefined reference to `pthread_create`
```
The `-lpthread` flag should be automatic. If building manually, ensure it's included:
```bash
gcc -std=c11 -o doip-server src/*.c -Iinclude -lpthread
```

**Permission denied on install:**
```bash
# Install requires root for /usr/sbin
sudo make install

# Or install to a staging directory
make install DESTDIR=/tmp/staging
```
